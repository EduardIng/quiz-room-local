# Technical Glossary - Технічний Глосарій

## A
**API (Application Programming Interface)** - Інтерфейс програмування додатків. Набір правил як програми спілкуються між собою.

**Auto-progression** - Автоматичний перехід. Система автоматично переходить до наступного питання без дії адміністратора.

**Autoplay Policy** - Правило автовідтворення. Браузерна політика що блокує автоматичне відтворення звуку якщо користувач ще не взаємодіяв зі сторінкою. Захищає від небажаного звуку. В проєкті обходиться через `.play().catch(() => {})` і кнопку ручного відтворення.

## B
**Backend** - Серверна частина додатку. Та частина що працює на сервері (MacBook).

**better-sqlite3** - Бібліотека Node.js для роботи з SQLite. Відрізняється синхронним API — запити виконуються одразу, без `async/await`.

**Build** - Збірка. Процес перетворення вихідного коду React у файли для браузера.

## C
**Category Mode** - Режим гри де перед кожним питанням один гравець (chooser) обирає між двома категоріями. Після вибору — відповідне питання задається всім. Chooser змінюється кожен раунд у порядку приєднання. Квіз зберігається у форматі `{ categoryMode: true, rounds: [...] }` замість `{ questions: [...] }`.

**Chooser** - Гравець що обирає категорію в Category Mode. Визначається ротацією через масив `playerJoinOrder`. Якщо не обрав за 15 секунд — сервер обирає випадково (`wasTimeout: true`).

**Commit** - Збереження змін в Git. Як "checkpoint" в грі.

**CORS (Cross-Origin Resource Sharing)** - Механізм безпеки браузера. Контролює які сайти можуть робити запити до сервера.

**Custom Hook** - Власний хук React. Функція що починається з `use` і інкапсулює логіку стану для повторного використання між компонентами. Наприклад: `useLang()`.

## D
**Dependencies** - Залежності. Бібліотеки які використовує наш проєкт (express, socket.io тощо).

**Drag and Drop API (HTML5 DnD)** - Вбудований браузерний API для перетягування елементів. Використовується в QuizCreator для зміни порядку питань. Елемент отримує `draggable="true"`, далі обробляються події `dragstart`, `dragover`, `drop`, `dragend`.

## E
**Express** - Веб-фреймворк для Node.js. Спрощує створення HTTP сервера.

**Event** - Подія. В Socket.IO - назва повідомлення яке надсилається між клієнтом і сервером.

## F
**Foreign Key** - Зовнішній ключ. Поле в таблиці БД що посилається на запис в іншій таблиці. У нашому проєкті: `session_id` в таблицях `results` і `question_stats` вказує на `id` в таблиці `sessions`.

**Frontend** - Клієнтська частина. Те що бачить користувач у браузері.

## G
**Git** - Система контролю версій. Зберігає всю історію змін коду.

**GitHub** - Хмарний сервіс для зберігання Git репозиторіїв.

**.gitignore** - Файл що перелічує шляхи які Git не відстежує (наприклад `node_modules/`, `data/`).

## H
**Hash routing** - Маршрутизація через `#` в URL. Наприклад `#/admin` або `#/stats`. Зміна частини після `#` не перезавантажує сторінку — React сам визначає який компонент показати.

**Hook** — Дивись **Custom Hook**.

**Host Controls** - Панель керування грою для ведучого (хоста). Доступна в QuizCreator після запуску кімнати. Кнопки: ▶ Start, ⏸ Pause, ▶ Resume, ⏭ Skip. Команди надсилаються через WebSocket подію `host-control`. Тільки socket що створив кімнату має права хоста (`roomHosts` Map на сервері).

## I
**i18n (Internationalization)** - Інтернаціоналізація. Підтримка кількох мов в інтерфейсі. Скорочення: між "i" та "n" — 18 літер. У проєкті реалізовано через `i18n.js` + хук `useLang()`.

## J
**JSON (JavaScript Object Notation)** - Формат даних. Текстовий файл для зберігання структурованих даних.

## L
**localStorage** - Сховище браузера для рядкових даних без терміну дії. Зберігається між закриттями вкладки і перезапусками браузера. Використовується для збереження вибраної мови (`lang`).

## N
**Node.js** - Середовище виконання JavaScript на сервері. Дозволяє запускати JS без браузера.

**npm** - Node Package Manager. Менеджер пакетів для Node.js.

**node-gyp** - Інструмент для компіляції нативних Node.js модулів (C/C++). Потрібен для `better-sqlite3`. Вимагає встановлених інструментів збірки (Xcode на macOS, build-essential на Linux).

## P
**playerJoinOrder** - Масив `socketId` в порядку приєднання до кімнати. Використовується в Category Mode для ротації chooser-а: `chooserIndex` вказує на поточний індекс у цьому масиві, збільшується на 1 після кожного раунду.

**PNG buffer** - Масив байтів що представляє PNG-зображення в пам'яті. Бібліотека `qrcode` генерує такий буфер, який сервер відправляє браузеру як відповідь `image/png`.

**Port** - Порт. Числова адреса для мережевих з'єднань (наш сервер: порт 8080).

**Push** - Відправка. Відправити локальні зміни на GitHub.

## Q
**qrcode** - npm-бібліотека для генерації QR-кодів. Використовується на сервері: `qrcode.toBuffer(url)` → PNG-зображення.

**QR-код (Quick Response code)** - Двовимірний штрих-код що кодує текст (зазвичай URL). Телефон сканує зображення і відкриває закодовану адресу. У проєкті кодує посилання для приєднання до кімнати: `http://<IP>:8080/?room=XXXXXX`.

**Query Parameter** - Параметр запиту в URL. Частина після `?`. Наприклад: `/?room=AB3C7D`. Читається через `new URLSearchParams(window.location.search).get('room')`.

## R
**Rate Limiting** - Обмеження частоти запитів. У проєкті: максимум 10 подій `submit-answer` на 30 секунд з одного socket-з'єднання. Реалізовано через `answerRateLimit` Map на сервері. Захищає від flood-атак і випадкових багів на клієнті.

**React** - JavaScript бібліотека для створення інтерфейсів користувача.

**Repository (repo)** - Репозиторій. Місце зберігання всього коду проєкту.

**roomHosts** - Map на сервері: `roomCode → socketId`. Зберігає який socket є хостом кожної кімнати. Перевіряється при кожній події `host-control` — тільки хост може керувати грою.

## S
**Singleton** - Патерн проєктування де клас має тільки один екземпляр. `db.js` експортує один об'єкт `QuizDatabase` — всі модулі що його імпортують отримують той самий екземпляр і ту саму відкриту БД.

**Socket.IO** - Бібліотека для real-time комунікації через WebSocket.

**SQLite** - Вбудована реляційна база даних що зберігається в одному файлі. Не потребує окремого сервера. Використовується для збереження історії сесій у `data/sessions.db`.

**State Machine** - Машина станів. Система що керує переходами між станами (WAITING → STARTING → QUESTION → ANSWER_REVEAL → LEADERBOARD → ENDED).

## T
**Transaction** - Транзакція. Група операцій з БД що виконуються як одне ціле: або всі успішно, або жодна. У `db.js` використовується `this.db.transaction()` щоб зберегти сесію, результати і статистику питань одним атомарним записом.

## U
**URLSearchParams** - Стандартний Web API для читання параметрів з URL. `new URLSearchParams(window.location.search).get('room')` повертає код кімнати якщо він переданий в адресі.

## V
**Vite** - Інструмент збірки для фронтенду. Замінює webpack. Дуже швидкий завдяки ES modules. Використовується для збірки React-додатку.

## W
**watch-room** - WebSocket подія для підключення як спостерігач (не гравець). Використовується ProjectorView: socket приєднується до кімнати, отримує всі broadcast-и, але не впливає на лічильник гравців, autostart або підрахунок відповідей. Зберігається в `observers` Map на сервері.

**WAL (Write-Ahead Logging)** - Режим журналювання SQLite. Покращує швидкість запису і дозволяє одночасне читання під час запису. Вмикається командою `PRAGMA journal_mode = WAL`.

**WebSocket** - Протокол двосторонньої комунікації між браузером і сервером в реальному часі.

## O
**object-fit** - CSS властивість що контролює як зображення масштабується всередині контейнера. `object-fit: contain` — зображення повністю вміщується без обрізки, зберігаючи пропорції. Використовується для `.question-image` щоб фото будь-якого розміру коректно відображались у відведеному блоці.

## PR
**Projector View** - Компонент `ProjectorView.jsx` (`#/screen`). Великий екран для телевізора або проектора в залі. Підключається через `watch-room` як спостерігач. Показує всі стани гри у великому форматі: QR-код на екрані очікування, питання з таймером, правильну відповідь, leaderboard. Не є гравцем — не впливає на логіку гри.
