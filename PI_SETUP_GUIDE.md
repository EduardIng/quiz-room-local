### 1. 🖥️ Raspberry Pi 5 (4GB)

Маленький комп'ютер розміром з кредитну картку (але товстіший). Зелена плата з чіпами, портами і 40-ма металевими штирками (GPIO) зверху.

- **Ціна:** ~€70–80 / ~1800–2000 Kč
- **Де купити:** RPishop.cz, Alza.cz, Amazon.de, BerryBase.de

### 2. 🔌 Блок живлення для Pi 5 (27W USB-C)

Білий (або чорний) адаптер з кабелем USB-C на кінці. Важливо: це НЕ звичайна зарядка від телефону — Pi 5 потребує 5V/5A (27 Ват). Звичайна зарядка дасть замало потужності і Pi буде глючити.

Роз'єм USB-C виглядає як маленький овальний конектор (такий самий як на нових MacBook та телефонах Android).

- **Ціна:** ~€12–15 / ~350–400 Kč
- **Де купити:** RPishop.cz, Alza.cz, Amazon.de
- **Що шукати:** "Official Raspberry Pi 5 Power Supply 27W"

### 3. 💾 Носій для операційної системи — Вибери ОДИН з трьох варіантів

На цьому носії буде операційна система для Pi — як жорсткий диск у звичайному комп'ютері. Є три варіанти. Тобі потрібен **ОДИН** — не всі три.

#### Варіант А: microSD картка (мінімум 16 ГБ, рекомендовано 32 ГБ)

Крихітна карта пам'яті розміром з ніготь великого пальця. Виглядає як маленький чорний прямокутник з позолоченими контактами з одного боку. Вставляється в спеціальний слот на нижній стороні Pi.

```
  microSD картка (вид зверху і знизу):

  Зверху:             Знизу:
  ┌──────────┐        ┌──────────┐
  │ SanDisk  │        │▓▓▓▓▓▓▓▓▓▓│  ← позолочені контакти
  │  32GB    │        │▓▓▓▓▓▓▓▓▓▓│
  │          │        │          │
  └───┐      │        └───┐      │
      └──────┘            └──────┘
       ↑ скошений кут — допомагає вставити правильною стороною
```

**Плюси:** найпростіше — Pi 5 завантажується з microSD "з коробки", нічого додатково налаштовувати не потрібно. Найдешевший варіант.

**Мінуси:** найповільніша швидкість; при постійній роботі (24/7 подіум) може зноситись за 1–2 роки.

- **Ціна:** ~€8–12 / ~200–300 Kč
- **Де купити:** Alza.cz, Amazon.de, будь-який магазин електроніки
- **Що шукати:** "microSD 32GB Class 10" або "microSD 32GB A2" (бренди: SanDisk, Samsung, Kingston)

#### Варіант Б: USB флешка (мінімум 16 ГБ, рекомендовано 32 ГБ)

Звичайна USB флешка — та сама, на якій ти, можливо, носиш документи. Маленький пластиковий корпус з USB конектором на одному кінці. Вставляється в USB порт на Pi.

```
  USB флешка (вид збоку):

  ┌──────────────────┐
  │  ████████████    ├═══╗
  │  Kingston 32GB   ├═══╝  ← USB-A конектор (плоский прямокутний)
  └──────────────────┘
```

**Плюси:** дешева, легко знайти в будь-якому магазині, швидша за microSD.

**Мінуси:** потрібне одноразове налаштування USB-завантаження на Pi (описано в Частині 2Б). Надійність залежить від якості флешки.

- **Ціна:** ~€8–15 / ~200–400 Kč
- **Де купити:** Alza.cz, Amazon.de, будь-який магазин електроніки
- **Що шукати:** "USB flash drive 32GB USB 3.0" (бренди: SanDisk, Samsung, Kingston)
- ⚠️ Обов'язково USB 3.0 (не 2.0) — різниця в швидкості величезна

#### Варіант В: USB SSD (мінімум 32 ГБ, рекомендовано 64–128 ГБ)

Зовнішній твердотільний диск — маленька коробочка (розміром з пачку жуйок або кредитну картку) з USB кабелем. Це той самий тип диска, що стоїть всередині сучасних ноутбуків, тільки в зовнішньому корпусі.

```
  USB SSD (вид зверху):

  ┌─────────────────────────┐
  │                         │
  │   Samsung T7  120GB     │    розмір: ~57 x 85 мм
  │                         │    (менше за кредитну картку)
  │                         │
  └────────────┐            │
               │  USB-C ════╡    ← кабель USB-C до USB-A
               └────────────┘
```

**Плюси:** найшвидший варіант (Pi завантажується за секунди), найнадійніший (розрахований на постійну роботу роками), великий об'єм.

**Мінуси:** найдорожчий, потрібне одноразове налаштування USB-завантаження на Pi (описано в Частині 2Б).

- **Ціна:** ~€25–50 / ~650–1300 Kč (за 120–128 ГБ)
- **Де купити:** Alza.cz, Amazon.de, Conrad.de
- **Що шукати:** "portable SSD USB-C" (бренди: Samsung T7, Kingston XS1000, SanDisk Extreme)
- ⚠️ Якщо SSD має USB-C конектор — потрібен кабель або адаптер USB-C → USB-A (бо USB порти Pi — це USB-A)

#### ⚡ Порівняльна таблиця

| | microSD картка | USB флешка | USB SSD |
|---|---|---|---|
| **Що це** | Крихітна карта пам'яті | Звичайна флешка | Зовнішній диск |
| **Розмір** | Як ніготь великого пальця | Як палець | Як пачка жуйок |
| **Швидкість** | Повільна | Середня | Дуже швидка |
| **Надійність** | Середня (зношується за 1–2 роки при постійній роботі) | Середня | Висока (працює роками) |
| **Ціна (32 ГБ)** | ~€8–12 / ~200–300 Kč | ~€8–15 / ~200–400 Kč | ~€25–50 / ~650–1300 Kč |
| **Простота** | Найпростіше — вставив і працює | Потрібне одноразове налаштування USB-завантаження | Потрібне одноразове налаштування USB-завантаження |
| **Для кого** | Настільний тест, починаючі | Золота середина | Повний подіум, постійна робота |

#### 🤔 Що вибрати?

- **Перший раз, просто спробувати** → microSD картка. Найпростіше, найдешевше, працює одразу.
- **Хочеш надійність, але бюджет обмежений** → USB флешка USB 3.0. Золота середина.
- **Будуєш постійний подіум** → USB SSD. Інвестиція в надійність і швидкість.

### 4. 🔗 Кабель Micro-HDMI → HDMI

Pi 5 має маленькі HDMI порти (micro-HDMI), а монітор — великий HDMI порт. Тобі потрібен кабель, де один кінець маленький (micro-HDMI), а другий — великий (звичайний HDMI).

Як відрізнити кінці:
- **Micro-HDMI** (маленький) — ширина ~6 мм, виглядає як мініатюрний HDMI
- **HDMI** (великий) — ширина ~14 мм, стандартний конектор для моніторів і телевізорів

Для настільного тесту потрібен **один** такий кабель. Для повного подіуму — два.

- **Ціна:** ~€8–12 / ~200–300 Kč
- **Де купити:** Alza.cz, Amazon.de, Conrad.de
- **Що шукати:** "Micro HDMI to HDMI cable" або "Micro HDMI кабель"

### 5. 🖥️ Монітор з HDMI входом

Будь-який монітор або телевізор з HDMI портом. Для настільного тесту підійде будь-який.

### 6. 🔘 GPIO кнопки (4 штуки)

Маленькі натискні кнопки (тактильні кнопки або кнопки для макетних плат). Виглядають як крихітні квадратні кнопочки з 4-ма металевими ніжками знизу. Натискаєш — контакт замикається, відпускаєш — розмикається.

Ідеально: 4 різних кольори (червона, синя, зелена, помаранчева) для чіткого розрізнення відповідей A, B, C, D.

- **Ціна:** ~€3–5 / ~80–150 Kč (за набір 10+ штук)
- **Де купити:** Alza.cz (пошук "taktilní tlačítko"), Amazon.de, Conrad.de
- **Що шукати:** "tactile push button" або "tact switch 6x6mm"

### 7. 🔌 Jumper проводи (мінімум 8 штук, female-to-male)

Тонкі кольорові проводи з конекторами на кінцях. Для нашого тесту потрібні **female-to-male** (з одного кінця "гніздо", з іншого "штирок"):
- **Female** (гніздо) — надівається на GPIO піни Pi
- **Male** (штирок) — вставляється в макетну плату або чіпляється до ніжки кнопки

Якщо маєш female-to-female (два "гнізда"), вони теж підійдуть — ніжки тактильних кнопок зазвичай є штирками (male).

- **Ціна:** ~€3–5 / ~80–150 Kč (за набір 40 штук)
- **Де купити:** Alza.cz, Amazon.de, Conrad.de
- **Що шукати:** "dupont jumper wires" або "propojovací kabely"

### 8. 🟩 Макетна плата / Breadboard (рекомендовано)

Біла або прозора пластикова дошка з рядами маленьких отворів. Дозволяє з'єднувати кнопки з проводами без пайки — просто вставляєш ніжки і проводи в отвори. Не є обов'язковою, але сильно спрощує підключення кнопок.

- **Ціна:** ~€3–5 / ~80–150 Kč
- **Де купити:** Alza.cz, Amazon.de, Conrad.de
- **Що шукати:** "breadboard 400 point" або "nepájivé kontaktní pole"

### 9. 🔷 Ethernet кабель

Кабель з двома однаковими широкими прозорими конекторами на кінцях (RJ45). Виглядає як товстіший телефонний кабель. Обидва кінці абсолютно ідентичні — не має значення який куди підключати.

- **Ціна:** ~€3–5 / ~80–150 Kč
- **Де купити:** Будь-який магазин електроніки, Alza.cz
- **Що шукати:** "Ethernet cable Cat5e" або "síťový kabel"

## Частина 1: Підготовка носія (SD-картка / USB флешка / USB SSD)

⏱ **Час: 15–20 хвилин**

Зараз ми запишемо операційну систему на твій обраний носій (microSD картку, USB флешку або USB SSD). Це як "встановити Windows", тільки для маленького комп'ютера Pi. Процес однаковий для всіх трьох типів носіїв — різниця тільки в тому, що саме ти підключаєш до Mac.

### Крок 1.1: Завантаж Raspberry Pi Imager

Шукай в Google: **"Raspberry Pi Imager download"**

Перший результат буде офіційний сайт raspberrypi.com. На сторінці знайди кнопку "Download for macOS" і натисни її. Завантажиться файл `.dmg`.

### Крок 1.2: Встанови Raspberry Pi Imager на Mac

1. Знайди завантажений файл (зазвичай в папці Downloads / Завантаження)
2. Двічі клікни на файл `.dmg` — відкриється вікно з іконкою програми і папкою Applications
3. Перетягни іконку Raspberry Pi Imager на папку Applications (це і є "встановлення" на Mac — просто перетягнути)
4. Закрий вікно `.dmg`
5. Відкрий Raspberry Pi Imager: натисни `Cmd + пробіл` (Spotlight), набери "Raspberry Pi Imager", натисни Enter

### Крок 1.3: Підключи носій до Mac

Перш ніж записувати, потрібно фізично підключити носій до Mac. Як саме — залежить від типу носія:

**📌 Якщо microSD картка:**
- Встав картку в Mac. Може знадобитись адаптер — маленька microSD картка вставляється в більший SD-адаптер (зазвичай йде в комплекті з карткою), який вставляється в слот SD на Mac або в USB кард-рідер.

**📌 Якщо USB флешка:**
- Встав флешку в будь-який USB порт на Mac. Просто вставив — і готово.

**📌 Якщо USB SSD:**
- Підключи SSD кабелем до USB порту Mac. Якщо SSD має USB-C конектор, а Mac має USB-A порти — потрібен адаптер або кабель USB-C → USB-A.

✅ **Що ти маєш бачити:** Finder може показати новий диск (або запитати чи відформатувати його). Не форматуй — Pi Imager зробить це автоматично.

### Крок 1.4: Вибери налаштування в Imager

Raspberry Pi Imager має три основні кнопки:

**1. "Choose Device" / "Вибрати пристрій":**
- Натисни → вибери **Raspberry Pi 5**

**2. "Choose OS" / "Вибрати ОС":**
- Натисни → вибери **Raspberry Pi OS (64-bit)**
- ⚠️ НЕ вибирай "Lite" — там немає графічного інтерфейсу, і Chromium (браузер) не запуститься
- Нам потрібна версія з Desktop (робочий стіл), бо Pi показуватиме гру в браузері на моніторі

**3. "Choose Storage" / "Вибрати носій":**
- Натисни кнопку → вибери свій носій зі списку
- Він буде показаний з назвою виробника і розміром (наприклад, "SanDisk 32GB" або "Samsung T7 120GB")
- ⚠️ Будь уважний — не вибери випадково диск свого Mac! Диск Mac зазвичай набагато більший (256 ГБ, 512 ГБ, 1 ТБ) і називається "Apple SSD" або "Macintosh HD"

### Крок 1.5: Налаштуй параметри (Advanced Settings)

Перед тим як записувати, натисни іконку ⚙️ (шестеренка) або кнопку "Edit Settings" — відкриються розширені налаштування. Заповни їх так:

| Параметр | Значення | Навіщо |
|----------|----------|--------|
| Set hostname | `pi-test` | Ім'я Pi в мережі |
| Enable SSH | ✅ (увімкнути), "Use password authentication" | Щоб керувати Pi з Mac без клавіатури |
| Set username | `pi` | ⚠️ Важливо: саме `pi`, не інше ім'я! Скрипти проєкту очікують цього імені |
| Set password | Придумай пароль, наприклад `quiz2026` | Запам'ятай його! Він потрібен для входу |
| Configure wireless LAN | Введи назву WiFi (SSID) і пароль | Щоб Pi підключився до мережі (пропусти якщо будеш використовувати тільки Ethernet кабель) |
| Wireless LAN country | `CZ` або `DE` | Вибери свою країну |
| Set locale settings | Timezone: `Europe/Prague` або `Europe/Berlin` | Для правильного часу |

Натисни "Save" / "Зберегти".

### Крок 1.6: Запиши на носій

1. Натисни "Write" / "Записати"
2. Підтверд, що згоден стерти всі дані на носії (там і так нічого потрібного немає)
3. З'явиться прогрес-бар — запис займе ~5–15 хвилин залежно від швидкості носія
   - microSD: ~5–10 хвилин
   - USB флешка: ~5–10 хвилин
   - USB SSD: ~3–5 хвилин (найшвидший)
4. Після запису програма автоматично перевірить дані (Verifying)

✅ **Що ти маєш бачити:** повідомлення "Write Successful" або "Запис успішний"

❌ **Якщо помилка запису:**
- **microSD:** спробуй іншу картку або інший USB-адаптер для картки
- **USB флешка:** спробуй інший USB порт на Mac або іншу флешку
- **USB SSD:** перевір кабель, спробуй інший USB порт

### Крок 1.7: Безпечно витягни носій

1. В Finder (панель ліворуч) знайди свій носій (він може називатись "boot", "bootfs" або ім'ям виробника)
2. Натисни іконку ⏏ (eject) поряд з назвою носія, або правий клік → "Eject" / "Витягнути"
3. Дочекайся повідомлення "Safe to remove" / "Можна витягати"
4. Фізично від'єднай носій:
   - **microSD:** витягни картку з кард-рідера або слоту Mac
   - **USB флешка:** витягни флешку з USB порту Mac
   - **USB SSD:** від'єднай кабель від Mac (або витягни кабель з SSD)

⚠️ **Не витягай носій без "Eject"!** Якщо витягнути під час запису або без безпечного витягнення — дані можуть пошкодитись і доведеться записувати знову.

✅ **Що ти маєш бачити:** носій зник з Finder, повідомлення що безпечно витягати

## Частина 2: Перше підключення Pi

⏱ **Час: 10 хвилин**

Зараз ми з'єднаємо всі фізичні компоненти і вперше увімкнемо Pi.

### Крок 2.1: Підключи носій з операційною системою до Pi

Цей крок залежить від того, який носій ти вибрав.

---

**📌 Якщо microSD картка:**

На нижній стороні Raspberry Pi 5 (протилежній від GPIO пінів) є слот для microSD. Встав картку:
- Позолоченими контактами **вниз** (до плати)
- Етикеткою **вгору**
- Вставляй доки не клацне на місце

```
  Raspberry Pi 5 (вид знизу):

  ┌───────────────────────────────────┐
  │                                   │
  │    (тут чіпи і компоненти)        │
  │                                   │
  │              ┌────────┐           │
  │              │ SD SLOT│ ← сюди    │
  │              └────────┘           │
  └───────────────────────────────────┘
                  ↑
           microSD вставляється сюди
           контактами ВНИЗ
```

📸 **Сфотографуй Pi з вставленою карткою**

➡️ **Переходь до Кроку 2.2**

---

**📌 Якщо USB флешка:**

Встав USB флешку в будь-який USB порт на Pi. Рекомендовано використовувати один з **синіх USB 3.0** портів (вони швидші).

```
  Raspberry Pi 5 (вид зверху, GPIO зверху):

            GPIO піни (40 штирків)
  ╔═══════════════════════════════════════╗
  ║                                       ║
  ║                                       ║
  ║   Ethernet                            ║
  ║   ┌─────┐   ┌─────┬─────┐            ║
  ║   │ RJ45│   │USB-A│USB-A│ ← чорні    ║
  ║   │     │   │ 2.0 │ 2.0 │   (повільні)║
  ║   └─────┘   ├─────┼─────┤            ║
  ║             │USB-A│USB-A│ ← СИНІ     ║
  ║             │ 3.0 │ 3.0 │   (швидкі) ║
  ║             └─────┴─────┘   СЮДИ! ★  ║
  ║                                       ║
  ╚═══════════════════════════════════════╝
```

⚠️ **Важливо:** Pi поки що не зможе завантажитись з USB флешки! За замовчуванням Pi 5 завантажується тільки з microSD. Щоб завантажитись з USB, потрібно одноразово змінити налаштування — це описано в **Частині 2Б**. Поки що просто встав флешку і переходь далі.

📸 **Сфотографуй Pi з підключеною флешкою**

➡️ **Переходь до Кроку 2.2**

---

**📌 Якщо USB SSD:**

Підключи USB SSD кабелем до одного з **синіх USB 3.0** портів на Pi. Сині порти — це ті, що в нижньому ряду USB портів (дивись схему вище для USB флешки).

⚠️ **Обов'язково в синій (USB 3.0) порт!** USB SSD працюватиме і в чорному (USB 2.0) порті, але буде дуже повільним — як Ferrari на першій передачі. Різниця в швидкості: USB 2.0 = ~35 МБ/с, USB 3.0 = ~350 МБ/с (в 10 разів швидше!).

⚠️ **Важливо:** Pi поки що не зможе завантажитись з USB SSD! За замовчуванням Pi 5 завантажується тільки з microSD. Щоб завантажитись з USB, потрібно одноразово змінити налаштування — це описано в **Частині 2Б**. Поки що просто підключи SSD і переходь далі.

📸 **Сфотографуй Pi з підключеним SSD**

➡️ **Переходь до Кроку 2.2**

### Крок 2.2: Підключи HDMI кабель

Raspberry Pi 5 має **два** порти micro-HDMI. Для нашого тесту підключи кабель до порту **HDMI 0** — це порт, який знаходиться **ближче до USB-C порту живлення** (вони поряд на одному боці Pi).

1. **Маленький кінець** кабелю (micro-HDMI) — в порт HDMI 0 на Pi
2. **Великий кінець** кабелю (звичайний HDMI) — в монітор
3. Увімкни монітор

### Крок 2.3: Підключи клавіатуру і мишку

Встав USB клавіатуру і мишку в будь-які USB порти на Pi. На Pi 5 є 4 USB порти:
- 2 сині (USB 3.0 — швидші)
- 2 чорні (USB 2.0 — повільніші)

Для клавіатури і мишки підійдуть будь-які з них.

### Крок 2.4: Підключи Ethernet (якщо використовуєш)

Якщо плануєш з'єднати Pi кабелем (а не через WiFi), встав Ethernet кабель:
- Один кінець — в Ethernet порт на Pi (широкий роз'єм RJ45, знаходиться на протилежному від USB-C кінці Pi)
- Другий кінець — в роутер або напряму в Mac (про це детальніше в Частині 3)

### Крок 2.5: Підключи живлення (ОСТАННІМ!)

⚠️ **Живлення підключай ОСТАННІМ!** Pi починає завантажуватись одразу як отримує живлення — немає кнопки ввімкнення. Тому спочатку підключи все інше, і тільки потім — кабель живлення.

1. Візьми блок живлення Pi 5 (27W USB-C)
2. Вставте USB-C конектор в порт живлення на Pi (він знаходиться на тому ж боці, що й HDMI порти)
3. Вставте блок живлення в розетку

📸 **Сфотографуй свій Pi зараз — з усіма підключеними кабелями. Якщо щось піде не так, фото допоможе розібратися.**

### Крок 2.6: Перше завантаження

Після підключення живлення:
1. На Pi засвітиться **червоний LED** (живлення є) і почне блимати **зелений LED** (йде завантаження)
2. На моніторі з'явиться кольоровий екран, потім текст завантаження
3. Перше завантаження займає ~1–2 хвилини (система налаштовується)
4. Після завантаження ти побачиш **робочий стіл** — шпалери з логотипом Raspberry Pi, панель завдань зверху

⚠️ **Якщо ти використовуєш USB флешку або USB SSD** і ще не налаштував USB-завантаження (Частина 2Б) — Pi НЕ завантажиться з USB. Це нормально! Тобі спочатку потрібна тимчасова microSD картка для налаштування. Перейди до Частини 2Б для інструкцій.

✅ **Що ти маєш бачити:** робочий стіл Pi з панеллю завдань зверху і значками

❌ **Якщо монітор темний (немає зображення):**
- Перевір що кабель HDMI підключений в порт HDMI 0 (ближче до порту живлення)
- Перевір що монітор увімкнений і вибрано правильне джерело (HDMI input)
- Спробуй інший HDMI кабель

❌ **Якщо тільки червоний LED (зелений не блимає):**
- Носій не читається — витягни, встав знову
- Якщо microSD — переконайся що клацнула на місце
- Якщо USB — спробуй інший USB порт
- Якщо не допомагає — повтори Частину 1 (перезапиши носій)

❌ **Якщо зелений LED блимає по 4 рази:**
- Проблема з носієм — спробуй перезаписати або використай інший носій

### Крок 2.7: Перевір інтернет-з'єднання

Якщо ти налаштував WiFi в Pi Imager, Pi вже має бути підключений до мережі. Перевіримо:

1. Клікни на іконку терміналу в панелі завдань Pi (чорний прямокутник з `>_`) — відкриється Terminal (термінал — це програма, де ти набираєш текстові команди)
2. Набери:

```bash
ping -c 3 google.com
```

3. Натисни Enter

✅ **Що ти маєш бачити:** три рядки відповідей, кожен починається з `64 bytes from...` і закінчується часом в мілісекундах. Це означає Pi має інтернет.

```
PING google.com (142.250.X.X) 56(84) bytes of data.
64 bytes from ...: icmp_seq=1 ttl=116 time=12.3 ms
64 bytes from ...: icmp_seq=2 ttl=116 time=11.8 ms
64 bytes from ...: icmp_seq=3 ttl=116 time=12.1 ms
```

❌ **Якщо `ping: unknown host` або `Network is unreachable`:**
- WiFi не підключений. Клікни на іконку мережі в панелі завдань (зверху справа), вибери свою WiFi мережу, введи пароль
- Або підключи Ethernet кабель до роутера

## Частина 2Б: Налаштування завантаження з USB (тільки для USB флешки та USB SSD)

⏱ **Час: 15–20 хвилин**

✅ **Якщо ти використовуєш microSD — ПРОПУСТИ цю частину!** Переходь одразу до Частини 3. Ця частина потрібна ТІЛЬКИ якщо ти хочеш завантажувати Pi з USB флешки або USB SSD.

### Проблема курки і яйця 🐔🥚

Тут є цікава проблема, яку легко зрозуміти на прикладі:

Уяви, що ти купив нову пральну машину, але інструкція з її налаштування — всередині замкненої машини. Щоб дістати інструкцію, потрібно відкрити машину. Щоб відкрити машину, потрібно прочитати інструкцію. Замкнене коло!

З Pi і USB та сама історія:
- **Pi за замовчуванням завантажується ТІЛЬКИ з microSD картки** — так він запрограмований на заводі
- **Щоб Pi завантажувався з USB**, потрібно змінити налаштування в "мозку" Pi (це називається EEPROM — маленький чіп пам'яті на платі Pi, який зберігає базові налаштування)
- **Щоб змінити це налаштування**, потрібно спочатку завантажити Pi... з microSD картки!

**Рішення:** ми використаємо **тимчасову** microSD картку. Запишемо на неї систему, завантажимо Pi, змінимо налаштування, вимкнемо Pi, витягнемо SD картку — і все! Тепер Pi запам'ятав, що можна завантажуватись з USB. Це налаштування зберігається **назавжди** — навіть якщо витягнути SD картку, вимкнути Pi з розетки, залишити на місяць. Pi пам'ятатиме.

Тимчасову SD картку потім можна використовувати для чого завгодно — вона більше не потрібна.

### Що тобі потрібно для цієї частини

- **Будь-яка microSD картка** (мінімум 8 ГБ) — тимчасова, тільки для зміни налаштування. Можна позичити у друга або купити найдешевшу.
- Твій основний USB носій (флешка або SSD) вже має бути записаний (Частина 1)

### Крок 2Б.1: Запиши тимчасову microSD картку

Повтори Частину 1 для тимчасової microSD картки:
1. Відкрий Raspberry Pi Imager на Mac
2. Choose Device → Raspberry Pi 5
3. Choose OS → Raspberry Pi OS (64-bit)
4. Choose Storage → вибери тимчасову microSD картку
5. Налаштуй параметри (те ж саме: hostname `pi-test`, SSH увімкнений, користувач `pi`, пароль)
6. Запиши і безпечно витягни

⚠️ Якщо ти вже завантажував Pi з microSD раніше (тобто у тебе вже є робоча SD картка) — можеш використати її замість запису нової. Просто встав її в Pi.

### Крок 2Б.2: Завантаж Pi з тимчасової SD картки

1. Встав тимчасову microSD картку в SD-слот Pi (знизу плати, контактами вниз)
2. Підключи монітор, клавіатуру, мережу (як в Частині 2)
3. Підключи живлення — Pi завантажиться з SD картки
4. Дочекайся робочого столу

### Крок 2Б.3: Відкрий термінал на Pi

На робочому столі Pi клікни на іконку терміналу (чорний прямокутник з `>_` в панелі завдань зверху).

Або підключись через SSH з Mac (якщо мережа вже працює):
```bash
ssh pi@pi-test.local
```

### Крок 2Б.4: Відкрий raspi-config

Набери в терміналі:

```bash
sudo raspi-config
```

Відкриється меню з синім фоном і сірими пунктами. Навігація: стрілки вгору/вниз для вибору пункту, Enter для входу, Tab для перемикання між пунктами і кнопками "Select"/"Back".

### Крок 2Б.5: Зміни порядок завантаження

1. Стрілками вибери **`6 Advanced Options`** → натисни Enter
2. Вибери **`A4 Boot Order`** → натисни Enter
3. Вибери **`B2 USB Boot`** → натисни Enter
   - Це означає: "спочатку спробуй завантажитись з USB, якщо не вийшло — спробуй SD картку"
4. З'явиться повідомлення "USB is default boot device" — натисни Enter (OK)

> raspi-config може запитати чи оновити EEPROM. Вибери **Yes**. Це безпечно — оновлення записує нові налаштування в прошивку Pi.

5. Натисни Tab щоб перейти на **"Finish"** → Enter
6. Система запитає "Would you like to reboot now?" — вибери **"Yes"** → Enter

Pi перезавантажиться. Це нормально — він ще завантажиться з тимчасової SD картки (бо вона все ще вставлена).

### Крок 2Б.6: Перевір що налаштування збережено

Після перезавантаження, в терміналі Pi набери:

```bash
sudo rpi-eeprom-config
```

✅ **Що ти маєш бачити** серед рядків виводу:

```
BOOT_ORDER=0xf14
```

Або:

```
BOOT_ORDER=0xf41
```

Обидва варіанти означають, що USB-завантаження увімкнене.

Пояснення (для допитливих): `BOOT_ORDER` — це число, де кожна цифра означає пристрій для завантаження. `4` = USB, `1` = SD картка, `f` = перезапуск циклу. `0xf14` означає: "спробуй SD, потім USB, повтори". `0xf41` означає: "спробуй USB, потім SD, повтори".

❌ **Якщо `BOOT_ORDER` не містить `4`:** повтори Крок 2Б.5.

### Крок 2Б.7: Вимкни Pi і заміни носій

1. Вимкни Pi правильно:
```bash
sudo shutdown now
```
2. Дочекайся поки зелений LED перестане блимати (5–10 секунд)
3. Від'єднай живлення
4. **Витягни тимчасову microSD картку** з слоту знизу Pi
5. **Підключи свій основний USB носій** (флешку або SSD) в USB порт Pi:
   - USB флешка → будь-який USB порт (краще синій USB 3.0)
   - USB SSD → обов'язково **синій USB 3.0** порт
6. ⚠️ Переконайся, що SD-слот **порожній** — інакше Pi завантажиться з SD картки, а не з USB

### Крок 2Б.8: Завантаж Pi з USB

1. Підключи живлення
2. Pi має завантажитись з USB носія!
3. Перше завантаження з USB займе ~1–3 хвилини

✅ **Що ти маєш бачити:** робочий стіл Pi — точно так само як при завантаженні з SD картки

❌ **Pi не завантажується (тільки червоний LED, зелений не блимає):**
- Перевір що SD-слот порожній
- Перевір що USB носій підключений правильно
- Спробуй інший USB порт (синій USB 3.0)
- Якщо USB SSD — перевір кабель
- Вставте тимчасову SD назад → завантажтесь → повторіть Крок 2Б.5

❌ **Зелений LED блимає, але монітор темний:**
- Перевір HDMI кабель
- Спробуй інший USB порт для носія

### Крок 2Б.9: Перевір з якого носія завантажився Pi

Щоб переконатися, що Pi дійсно працює з USB, а не з microSD, відкрий термінал і набери:

```bash
lsblk
```

Шукай рядок з `/` (кореневий розділ):
- Якщо `mmcblk0p2` — це microSD (щось пішло не так, Pi завантажився з SD)
- Якщо `sda2` — це USB флешка або USB SSD (все правильно!)

✅ `sda2` змонтований як `/` — Pi працює з USB. microSD більше не потрібна!

⚠️ **Збережи microSD картку!** Не форматуй і не викидай її. Якщо щось піде не так з USB носієм — ти завжди можеш завантажити Pi з microSD і полагодити проблему.

### 🎉 Готово! USB-завантаження налаштоване!

Тепер Pi завантажуватиметься з USB-носія автоматично. Це налаштування зберігається **назавжди** в EEPROM чіпі Pi — навіть при повному вимкненні.

📸 **Сфотографуй Pi з підключеним USB носієм — все працює!**

## Частина 3: Підключення через SSH

⏱ **Час: 15–20 хвилин**

SSH (Secure Shell) — це як пульт дистанційного керування комп'ютером через мережу. Ти набираєш команди на Mac, а Pi їх виконує. Це дозволяє керувати Pi без клавіатури та монітора, підключених до нього.

Є три способи з'єднати Mac та Pi. Вибери той, що підходить тобі:

---

### Варіант А: Ethernet кабель напряму до Mac

Найпростіший варіант — з'єднай Mac і Pi кабелем напряму, без роутера.

**Фізичне підключення:**
1. Ethernet кабель: один кінець в Pi, другий в Mac
2. Mac mini: Ethernet порт на задній панелі
3. MacBook Air: потрібен USB-C → Ethernet адаптер (якщо немає вбудованого порту)

**Налаштування Internet Sharing на Mac (щоб Pi отримав IP адресу):**

IP адреса — це як поштова адреса комп'ютера в мережі. Кожен пристрій має свою унікальну адресу, щоб інші пристрої знали куди слати дані. Виглядає як 4 числа, розділені крапками: наприклад, `192.168.2.1`.

1. На Mac відкрий **System Settings** (Налаштування системи): натисни `Cmd + пробіл`, набери "System Settings", натисни Enter
2. В пошуковому полі набери **"Sharing"** або **"Internet Sharing"**
3. Знайди **Internet Sharing** (Спільний інтернет)
4. "Share your connection from" → вибери **Wi-Fi** (або те з'єднання, через яке Mac має інтернет)
5. "To computers using" → постав галочку на **Ethernet** (або "USB Ethernet")
6. Увімкни Internet Sharing (повзунок або галочка)
7. Підтверд у діалоговому вікні

Тепер Mac роздає інтернет через Ethernet кабель, і Pi автоматично отримає IP адресу.

**Знаходимо IP адресу Pi:**

На Mac відкрий Terminal (термінал — програма для текстових команд, як і на Pi):
- Натисни `Cmd + пробіл`, набери "Terminal", натисни Enter

Набери:
```bash
arp -a
```

Шукай рядок з адресою, що починається на `192.168.2.` — це буде IP адреса Pi. Наприклад: `192.168.2.2`.

Або спробуй:
```bash
ping pi-test.local
```

✅ **Що ти маєш бачити:** відповіді з IP адресою Pi, наприклад `PING pi-test.local (192.168.2.2)`

---

### Варіант Б: Обидва через WiFi

Mac і Pi підключені до однієї WiFi мережі (ти налаштував WiFi в Pi Imager в Частині 1).

**Знаходимо IP адресу Pi:**

На Pi (в терміналі Pi, з клавіатури підключеної до Pi) набери:
```bash
hostname -I
```

✅ **Що ти маєш бачити:** IP адресу, наприклад `192.168.1.105`. Запиши цю адресу — вона потрібна далі!

Якщо не маєш доступу до Pi (немає клавіатури), спробуй на Mac:
```bash
ping pi-test.local
```

Або:
```bash
arp -a | grep -i "pi"
```

---

### Варіант В: Через роутер (Ethernet кабелі)

І Mac, і Pi підключені до одного роутера кабелями.

**Знаходимо IP адресу Pi:**
- Так само як у Варіанті Б: `hostname -I` на Pi або `ping pi-test.local` на Mac
- Або подивись в адмін-панелі роутера (зазвичай `192.168.1.1` в браузері) — знайди пристрій з ім'ям `pi-test`

---

### Підключення SSH (однаково для всіх варіантів)

Тепер, коли ти знаєш IP адресу Pi, підключись через SSH з Mac.

На Mac, в Terminal, набери (заміни `АДРЕСА` на реальну IP адресу Pi):

```bash
ssh pi@АДРЕСА
```

Наприклад:
```bash
ssh pi@192.168.1.105
```

Або, якщо `pi-test.local` працює:
```bash
ssh pi@pi-test.local
```

**Що станеться при першому підключенні:**

Ти побачиш попередження:
```
The authenticity of host '192.168.1.105' can't be established.
ED25519 key fingerprint is SHA256:xxxxxxxxxxx
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Це нормально! Pi каже: "Привіт, ми раніше не спілкувались. Ти впевнений що хочеш підключитись?" Набери `yes` і натисни Enter.

Далі введи пароль, який ти встановив в Pi Imager (наприклад, `quiz2026`). Коли набираєш пароль — символи НЕ відображаються на екрані. Це нормально — просто набирай і натисни Enter.

✅ **Що ти маєш бачити:** запрошення (prompt) змінилось! Замість імені твого Mac ти бачиш щось на зразок:

```
pi@pi-test:~ $
```

Це означає: ти зараз всередині Pi, і всі команди, які ти набираєш, виконуються на Pi, а не на Mac.

📸 **Сфотографуй екран — ти підключився до Pi через SSH!**

❌ **"Connection refused":** SSH не увімкнений. Повтори Частину 1 і переконайся, що Enable SSH був увімкнений в налаштуваннях Pi Imager.

❌ **"No route to host" або "Connection timed out":** Неправильна IP адреса або Mac і Pi не в одній мережі. Перевір IP ще раз.

❌ **"Permission denied":** Неправильний пароль. Спробуй ще раз, набирай уважно (пам'ятай, пароль не відображається).

## Частина 4: Налаштування Mac як сервера

⏱ **Час: 20–30 хвилин**

### Як все працює — загальна картина

Перш ніж почнемо — давай зрозуміємо що до чого. Ось як влаштована наша система:

```
  ╔═══════════════╗                              ╔═══════════════╗
  ║   Mac mini    ║       Одна мережа            ║ Raspberry Pi  ║
  ║  або MacBook  ║◄═══════════════════════════►║     5         ║
  ║               ║  (WiFi, Ethernet, або        ║               ║
  ║ ┌───────────┐ ║   напряму через кабель)      ║ ┌───────────┐ ║
  ║ │ Сервер    │ ║                              ║ │ Chromium  │ ║
  ║ │ квізу     │ ║     Дані гри (питання,       ║ │ (браузер) │ ║
  ║ │ порт 8080 │ ║◄────відповіді, рахунок)────►║ │ PlayerView│ ║
  ║ └───────────┘ ║                              ║ └───────────┘ ║
  ║               ║                              ║               ║
  ║ 192.168.X.X   ║                              ║ ┌───────────┐ ║
  ║               ║     Натискання кнопок        ║ │ GPIO      │ ║
  ║               ║◄────────────────────────────║ │ кнопки    │ ║
  ╚═══════════════╝                              ║ └───────────┘ ║
                                                 ║ 192.168.X.Y   ║
                                                 ╚═══════════════╝
```

**Mac** = мозок системи. На ньому працює сервер квізу — програма, яка зберігає питання, рахує бали, керує грою.

**Pi** = екран + кнопки. Pi просто показує те, що сервер йому каже, і надсилає натискання кнопок назад на сервер. Pi не "думає" сам.

Вони спілкуються через мережу (WiFi або Ethernet кабель) на порту 8080 (порт — це як номер квартири в будинку: IP адреса — це адреса будинку, а порт — конкретна квартира).

### Для Mac mini

**Крок 4.1: Відкрий Terminal на Mac**

Terminal — це програма, де ти набираєш текстові команди. Три способи відкрити:

1. **Spotlight (найшвидший):** натисни `Cmd + пробіл`, набери "Terminal", натисни Enter
2. **Finder:** відкрий Finder → Applications → Utilities → Terminal
3. **Launchpad:** натисни іконку Launchpad в Dock (ракета), знайди Terminal

✅ **Що ти маєш бачити:** чорне або біле вікно з текстом і курсором, що блимає. Щось типу:

```
einhorn@mac-mini ~ %
```

**Крок 4.2: Перевір Node.js**

Node.js — це програма, яка запускає наш сервер квізу. Перевіримо чи вона є:

```bash
node --version
```

✅ **Що ти маєш бачити:** номер версії, наприклад `v20.11.0` або `v22.3.0`. Важливо щоб було v18 або вище.

❌ **Якщо `command not found`:**

Node.js не встановлений. Встановимо через Homebrew (менеджер пакетів для Mac).

Спочатку встановимо Homebrew. Шукай в Google: **"Homebrew install"** — на першій сторінці буде brew.sh з командою встановлення. Або набери в Terminal:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Установка Homebrew займе ~2–5 хвилин. Дотримуйся інструкцій на екрані (може попросити пароль Mac).

Після установки Homebrew, встанови Node.js:

```bash
brew install node
```

Перевір ще раз:
```bash
node --version
```

**Крок 4.3: Перевір git**

Git — це програма для завантаження коду з інтернету (і не тільки). Перевіримо:

```bash
git --version
```

✅ **Що ти маєш бачити:** `git version 2.XX.X`

❌ **Якщо `command not found`:**

```bash
xcode-select --install
```

Відкриється діалогове вікно — натисни "Install" і дочекайся завершення (~5–10 хвилин).

**Крок 4.4: Завантаж проєкт**

```bash
git clone https://github.com/EduardIng/quiz-room-local.git
```

Ця команда завантажує весь проєкт з GitHub (сховище коду в інтернеті) на твій Mac.

✅ **Що ти маєш бачити:** рядки з прогресом завантаження, завершується повідомленням типу `Cloning into 'quiz-room-local'...`

```bash
cd quiz-room-local
```

Ця команда переходить в папку проєкту (як відкрити папку в Finder, тільки в Terminal).

**Крок 4.5: Встанови залежності**

```bash
npm install
```

Ця команда завантажує всі бібліотеки, які потрібні серверу. Займає ~1–2 хвилини. На екрані будуть рядки з назвами пакетів.

✅ **Що ти маєш бачити:** наприкінці — повідомлення `added XXX packages in Xs` без червоних помилок

Тепер зберемо фронтенд (інтерфейс гравця):

```bash
cd frontend && npm install && npm run build && cd ..
```

Це одна довга команда, яка:
1. Переходить в папку frontend (`cd frontend`)
2. Встановлює залежності фронтенду (`npm install`)
3. Будує оптимізовану версію інтерфейсу (`npm run build`) — результат зберігається в `frontend/build/`
4. Повертається назад в основну папку (`cd ..`)

⚠️ Збірка фронтенду може зайняти ~30–60 секунд. Дочекайся завершення.

✅ **Що ти маєш бачити:** наприкінці — рядки типу `✓ built in X.XXs` без помилок

**Крок 4.6: Запусти сервер**

```bash
npm start
```

✅ **Що ти маєш бачити:** красивий банер:

```
╔════════════════════════════════════════╗
║     Quiz Room Local - Запущено!        ║
╠════════════════════════════════════════╣
║  Локально:  http://localhost:8080     ║
║  Мережа:    http://192.168.X.X:8080  ║
║                                        ║
║  Планшети: підключаються автоматично   ║
╚════════════════════════════════════════╝
```

Зверни увагу на рядок "Мережа:" — там буде IP адреса твого Mac. **Запиши цю адресу!** Вона потрібна для підключення Pi.

❌ **Якщо `Error: listen EADDRINUSE :::8080`:** порт 8080 вже зайнятий іншою програмою. Знайди що його використовує:
```bash
lsof -i :8080
```
Закрий ту програму або перезавантаж Mac.

❌ **Якщо `MODULE_NOT_FOUND`:** ти забув запустити `npm install`. Зупини сервер (`Ctrl+C`) і запусти:
```bash
npm install && npm start
```

**Крок 4.7: Знайди IP адресу Mac**

IP адреса Mac показана в банері сервера (рядок "Мережа"). Але якщо потрібно дізнатися її окремо:

Це покаже IP адресу WiFi. Для Ethernet:

1. Відкрий System Settings → Network
2. Вибери активне з'єднання (WiFi або Ethernet)
3. IP адреса буде написана поруч

Важливо: тобі потрібна адреса, що починається на `192.168.` або `10.` — це адреса в локальній мережі. Адреса `127.0.0.1` (або `localhost`) — це адреса комп'ютера "для себе", інші пристрої за нею не підключаться.

**Крок 4.8: Перевір в браузері**

Не закриваючи Terminal з сервером, відкрий Safari або Chrome на Mac:

Перейди на адресу:

## Частина 5: Встановлення софту на Pi

⏱ **Час: 15–20 хвилин**

Зараз ми встановимо на Pi все необхідне для показу гри та роботи GPIO кнопок.

### Що ми встановлюємо і навіщо

На Pi нам потрібні тільки дві речі:
1. **Chromium** (браузер) — щоб показувати інтерфейс гравця на моніторі. Він вже встановлений в Pi OS Desktop! ✅
2. **Python бібліотеки** — щоб GPIO кнопки надсилали натискання на сервер. Потрібно встановити.

### Крок 5.1: Підключись до Pi через SSH

Якщо ти ще не підключений — на Mac в Terminal:
```bash
ssh pi@АДРЕСА_PI
```

### Крок 5.2: Оновлення системи

```bash
sudo apt-get update -q
```

Ця команда оновлює список доступних програм (`sudo` означає "виконай з правами адміністратора" — як "Запустити від імені адміністратора" в Windows).

```bash
sudo apt-get upgrade -y -q
```

Ця команда оновлює всі встановлені програми до найновіших версій. Може зайняти ~5–10 хвилин.

### Крок 5.3: Встанови Python залежності для GPIO

```bash
sudo apt-get install -y python3-pip
```

Це встановить pip — менеджер пакетів для Python (як npm для Node.js).

Тепер встановимо бібліотеки для GPIO кнопок:

```bash
pip3 install --break-system-packages "python-socketio[client]"
```

⚠️ **Прапорець `--break-system-packages` обов'язковий** на Raspberry Pi OS Bookworm. Без нього pip відмовиться встановлювати пакети і покаже помилку про "externally-managed-environment". Це не небезпечно — прапорець просто дозволяє pip встановити пакет в системну директорію Python.

Ця команда встановлює `python-socketio` — бібліотеку, яка дозволяє Python спілкуватись з сервером квізу через мережу. Пакет `websocket-client` (потрібен для з'єднання) встановиться автоматично як залежність.

✅ **Що ти маєш бачити:** рядки завантаження пакетів, наприкінці — `Successfully installed ...`

**Перевіримо GPIO бібліотеку:**

На Pi 5 з Raspberry Pi OS Bookworm бібліотека для GPIO (`rpi-lgpio`, яка надає сумісний інтерфейс `RPi.GPIO`) має бути вже встановлена. Перевіримо:

```bash
python3 -c "import RPi.GPIO; print('GPIO OK')"
```

✅ **Що ти маєш бачити:** `GPIO OK`

❌ **Якщо `ModuleNotFoundError`:**
```bash
sudo apt-get install -y python3-rpi-lgpio
```

⚠️ **Важливо для Pi 5:** НЕ встановлюй `RPi.GPIO` через pip (`pip3 install RPi.GPIO`). Цей пакет не підтримує чіп RP1 в Pi 5. Замість нього на Pi 5 використовується `rpi-lgpio`, який вже є в Pi OS або встановлюється через `apt-get` (як показано вище).

### Крок 5.4: Завантаж проєкт на Pi

Нам потрібен проєкт на Pi, бо там лежить файл `gpio-service.py` (програма, яка зчитує натискання кнопок).

```bash
git clone https://github.com/EduardIng/quiz-room-local.git /home/pi/quiz-room-local
```

✅ **Що ти маєш бачити:** прогрес завантаження, без помилок

❌ **Якщо `destination path already exists`:** проєкт вже завантажений раніше. Це ОК, продовжуй далі.

### Крок 5.5: Налаштуй адресу сервера в GPIO скрипті

GPIO скрипт за замовчуванням підключається до `localhost:8080` (тобто до сервера на самому Pi). Але наш сервер на Mac! Потрібно змінити адресу.

```bash
nano /home/pi/quiz-room-local/pi-setup/gpio-service.py
```

`nano` — це текстовий редактор в терміналі. Він виглядає як текстовий файл з підказками внизу екрану. Навігація: стрілки на клавіатурі.

Знайди рядок (він близько до початку файлу, рядок 19):
```python
SERVER_URL = 'http://localhost:8080'
```

Зміни `localhost` на IP адресу твого Mac (ту, що ти записав в Частині 4):
```python
SERVER_URL = 'http://192.168.X.X:8080'
```

Як зберегти і вийти з nano:
1. Натисни `Ctrl + X` (вийти)
2. Nano запитає "Save modified buffer?" — натисни `Y` (так)
3. Натисни `Enter` (підтвердити ім'я файлу)

✅ **Що ти маєш бачити:** ти повернувся в звичайний термінал

📸 **Зроби фото екрану перед збереженням — щоб бачити що саме ти змінив**

### ⚠️ Аудит install.sh — відомі проблеми

Скрипт `install.sh` в `pi-setup/` призначений для повного автоматичного налаштування, але має кілька проблем на Pi 5 з Bookworm. Для настільного тесту ми встановлювали все вручну (кроки вище), тому ці проблеми нас не стосуються. Але якщо ти захочеш запустити `install.sh` для повного налаштування подіуму в майбутньому — ось що потрібно знати:

1. ⚠️ **`pip3 install RPi.GPIO`** (рядок 22 install.sh) — цей пакет **не працює на Pi 5**. На Pi 5 потрібен `rpi-lgpio` (вже є в Pi OS Bookworm). Видали `RPi.GPIO` з команди pip install.

2. ⚠️ **`pip3 install` без `--break-system-packages`** (рядок 22) — на Bookworm (Python 3.11+) pip не дозволяє встановлювати пакети без цього прапорця. Потрібно додати `--break-system-packages`.

3. ⚠️ **Шляхи `/home/pi/quiz-room-local`** (рядки 25, 28, 35, 40, 46) — жорстко прописані для користувача `pi`. Якщо ти створив іншого користувача — шляхи не працюватимуть.

4. ⚠️ **Autostart через `.config/autostart/`** (рядки 34–43) — потребує Desktop варіант Pi OS. На Lite (без робочого столу) не працюватиме.

## Частина 6: Підключення GPIO кнопок

⏱ **Час: 25–30 хвилин** (найдовша частина — не поспішай!)

⚠️ **БЕЗПЕКА:** Завжди підключай проводи коли Pi **ВИМКНЕНИЙ** або принаймні коли ти точно знаєш до якого піна підключаєш. Підключення проводу до неправильного піна може пошкодити Pi або кнопку. Ніколи не підключай провід навмання.

### Що таке GPIO

GPIO (General Purpose Input/Output) — це 40 металевих штирків (пінів) на верхній частині плати Pi. Через них Pi може "відчувати" фізичний світ: чи натиснута кнопка, який сигнал йде. В нашому випадку — Pi дізнається, коли ти натискаєш кнопку відповіді.

### Схема пінів GPIO

Ось повна схема GPIO пінів Raspberry Pi 5. Дивись зверху на плату, USB порти знизу:

```
            ╔═══════════════════════════════════════════════════╗
            ║           RASPBERRY PI 5 — GPIO HEADER           ║
            ║      (вид зверху, USB порти знизу)               ║
            ╠═══════════════════════════════════════════════════╣
            ║                                                   ║
            ║   (1)  3V3    ●  ●   5V     (2)                  ║
            ║   (3)  GPIO2  ●  ●   5V     (4)                  ║
            ║   (5)  GPIO3  ●  ●   GND    (6)  ◄── земля       ║
            ║   (7)  GPIO4  ●  ●   GPIO14 (8)                  ║
            ║   (9)  GND    ●  ●   GPIO15 (10) ◄── земля       ║
            ║  (11) GPIO17  ★  ●   GPIO18 (12) ★ = Кнопка A    ║
            ║  (13) GPIO27  ★  ●   GND    (14) ★ = Кнопка B    ║
            ║  (15) GPIO22  ★  ★   GPIO23 (16) ★ = Кнопки C, D ║
            ║  (17)  3V3    ●  ●   GPIO24 (18)                 ║
            ║  (19) GPIO10  ●  ●   GND    (20) ◄── земля       ║
            ║  (21)  GPIO9  ●  ●   GPIO25 (22)                 ║
            ║  (23) GPIO11  ●  ●   GPIO8  (24)                 ║
            ║  (25)  GND    ●  ●   GPIO7  (26)                 ║
            ║  (27)  GPIO0  ●  ●   GPIO1  (28)                 ║
            ║  (29)  GPIO5  ●  ●   GND    (30)                 ║
            ║  (31)  GPIO6  ●  ●   GPIO12 (32)                 ║
            ║  (33) GPIO13  ●  ●   GND    (34)                 ║
            ║  (35) GPIO19  ●  ●   GPIO16 (36)                 ║
            ║  (37) GPIO26  ●  ●   GPIO20 (38)                 ║
            ║  (39)  GND    ●  ●   GPIO21 (40)                 ║
            ║                                                   ║
            ╠═══════════════════════════════════════════════════╣
            ║                    USB порти ↓                     ║
            ╚═══════════════════════════════════════════════════╝

            ★ — піни для наших кнопок
            ◄── земля — піни GND (заземлення)
```

### Наші 4 кнопки

| Кнопка | Відповідь | Колір (рекомендований) | GPIO пін (BCM) | Фіз. пін | GND пін (рекомендований) |
|--------|-----------|----------------------|----------------|----------|--------------------------|
| A | Перша відповідь | 🔴 Червоний | GPIO17 | Пін 11 | Пін 9 |
| B | Друга відповідь | 🔵 Синій | GPIO27 | Пін 13 | Пін 14 |
| C | Третя відповідь | 🟢 Зелений | GPIO22 | Пін 15 | Пін 14 або 20 |
| D | Четверта відповідь | 🟠 Помаранчевий | GPIO23 | Пін 16 | Пін 20 |

**Що таке BCM і фізичний пін?**
- **Фізичний пін** — це просто номер штирка зліва направо, зверху вниз (1, 2, 3... 40). Рахуємо штирки.
- **BCM (Broadcom)** — це назва GPIO в програмному коді. Наш скрипт використовує BCM нумерацію.
- Наприклад: фізичний пін 11 = GPIO17 в BCM. Це один і той самий штирок, просто два способи на нього посилатись.

**Що таке GND?**
- GND (Ground / Земля) — це "мінус" електричного кола. Кнопка працює так: один провід від кнопки йде на GPIO пін, другий — на GND. Коли натискаєш кнопку, GPIO пін з'єднується з GND, і Pi це "відчуває".
- На Pi є кілька GND пінів: 6, 9, 14, 20, 25, 30, 34, 39. Всі вони однакові — можна використовувати будь-який. Кілька кнопок можуть ділити один GND пін.

### Підключення кнопок

⚠️ **Переконайся, що Pi вимкнений (кабель живлення від'єднаний) перед підключенням проводів!**

#### З макетною платою (Breadboard) — рекомендовано

Макетна плата має центральну "канавку" (gap), яка розділяє її на дві половини. Кнопка ставиться так, щоб її ніжки були по обидва боки канавки:

```
  Макетна плата (breadboard)
  ┌───────────────────────────┐
  │ ● ● ● ● │ ● ● ● ● ● ● ● │
  │ ● ● ● ● │ ● ● ● ● ● ● ● │
  │ ● ● [K]━━━━[K] ● ● ● ● ● │   ← кнопка перетинає канавку
  │ ● ● [K]━━━━[K] ● ● ● ● ● │
  │ ● ● ● ● │ ● ● ● ● ● ● ● │
  └───────────────────────────┘
        ↑                ↑
      GPIO              GND
      провід            провід
```

**Кнопка A (червона):**
1. Встав кнопку в макетну плату, щоб вона перетинала канавку
2. Візьми перший провід. Один кінець (female) — на **пін 11** Pi (GPIO17). Другий кінець (male) — в отвір макетної плати на тому ж рядку, що й одна ніжка кнопки
3. Візьми другий провід. Один кінець (female) — на **пін 9** Pi (GND). Другий кінець (male) — в отвір на рядку з **протилежною** ніжкою кнопки

📸 **Сфотографуй кнопку A після підключення**

**Кнопка B (синя):**
1. Нова кнопка на макетній платі
2. Провід 1: ніжка кнопки → **пін 13** Pi (GPIO27)
3. Провід 2: протилежна ніжка → **пін 14** Pi (GND)

📸 **Сфотографуй кнопку B після підключення**

**Кнопка C (зелена):**
1. Нова кнопка
2. Провід 1: ніжка кнопки → **пін 15** Pi (GPIO22)
3. Провід 2: протилежна ніжка → **пін 20** Pi (GND)

📸 **Сфотографуй кнопку C після підключення**

**Кнопка D (помаранчева):**
1. Нова кнопка
2. Провід 1: ніжка кнопки → **пін 16** Pi (GPIO23)
3. Провід 2: протилежна ніжка → **пін 20** Pi (GND)

📸 **Сфотографуй кнопку D після підключення**

#### Без макетної плати — напряму

Якщо немає breadboard, можна підключити проводи напряму до ніжок кнопок:
1. Візьми female-to-female провід
2. Один кінець надінь на ніжку кнопки (обережно, щоб тримався)
3. Другий кінець надінь на GPIO пін на Pi
4. Те ж саме для другого проводу (ніжка → GND пін)

Це менш надійно (проводи можуть спадати), але для тесту працює.

### Типові помилки і як їх виправити

| Проблема | Симптом | Рішення |
|----------|---------|---------|
| Провід на неправильному піні | Натискаєш A, спрацьовує B (або нічого) | Перерахуй піни від пін 1 (3V3) — він в кутку ближче до SD слоту |
| Провід погано тримається | Кнопка іноді працює, іноді ні | Надінь конектор щільніше, або використай breadboard |
| Обидва проводи на одній ніжці кнопки | Кнопка взагалі не працює | У кнопки 4 ніжки — GPIO і GND мають бути на ПРОТИЛЕЖНИХ ніжках |
| Кнопка повернута на 90° | Не працює | Поверни кнопку на 90° — внутрішні контакти з'єднані парами |

### Крок 7.1: Переконайся, що сервер працює на Mac

Якщо ти вимикав Pi для підключення кнопок — увімкни його знову. Дочекайся завантаження робочого столу (~1–2 хвилини).

### Крок 7.3: Відкрий браузер на Pi

У тебе є два варіанти:

**Варіант А — з клавіатури і мишки Pi (простіший):**
1. На робочому столі Pi знайди іконку **Chromium** (або Web Browser) в панелі завдань зверху, або в меню додатків
2. Клікни — відкриється браузер
3. В адресному рядку набери адресу сервера (де `192.168.X.X` — IP адреса твого Mac):

**Варіант Б — через SSH з Mac (якщо немає клавіатури на Pi):**

Підключись по SSH і запусти Chromium з командного рядка. Для цього потрібно мати доступ до дисплея Pi:

```bash
DISPLAY=:0 /usr/bin/chromium http://192.168.X.X:8080 &
```

> Примітка: пакет chromium на Raspberry Pi OS Trixie більше **не** надає
> команди `chromium-browser` — використовуй абсолютний шлях
> `/usr/bin/chromium`.

Якщо з'явиться помилка доступу до дисплея, спробуй:
```bash
export DISPLAY=:0
export XAUTHORITY=/home/admin/.Xauthority   # або /home/pi/.Xauthority — залежить від користувача
/usr/bin/chromium http://192.168.X.X:8080 &
```

⚠️ Варіант Б складніший і може не працювати на Wayland (нова графічна система в Bookworm). Рекомендуємо Варіант А для настільного тесту.

```
┌─────────────────────────────┐
│                             │
│   Очікуємо ведучого...      │
│   Гра розпочнеться          │
│   автоматично               │
│                             │
└─────────────────────────────┘
```

Це означає: Pi підключився до сервера, але ведучий ще не запустив квіз. Все правильно!

❌ **Білий або порожній екран:**
- Неправильна IP адреса Mac — перевір і спробуй ще раз
- Сервер не запущений на Mac — перевір Terminal на Mac

❌ **"Не вдається відкрити сторінку" / "This site can't be reached":**

### Крок 9.4: Вибір категорії

- На екрані з'являться **2 варіанти категорій**
- Тобі потрібно вибрати: натисни **кнопку A** (перший варіант) або **кнопку B** (другий варіант)

Після вибору на екрані з'явиться "Обрана категорія: [назва]" і відлік 4 секунди перед першим питанням.

### Крок 9.5: Відповідай на питання

На екрані з'явиться питання з 4 варіантами відповідей і таймер зворотного відліку (30 секунд за замовчуванням).

Варіанти відповідей пронумеровані — кнопки відповідають варіантам:

| Кнопка | Варіант відповіді |
|--------|-------------------|
| A (GPIO17) | Перший варіант |
| B (GPIO27) | Другий варіант |
| C (GPIO22) | Третій варіант |
| D (GPIO23) | Четвертий варіант |

Натисни кнопку, яка відповідає правильному (або будь-якому) варіанту.

✅ **На екрані Pi:** "Відповідь надіслана!" і "Чекаємо на інших гравців..."

### Крок 9.6: Результат відповіді

Після завершення таймера (або коли всі гравці відповіли) з'являється екран результатів:

- **"Правильно!"** + кількість балів — якщо відповідь була правильною
- **"Неправильно!"** — якщо відповідь була неправильною
- Показується правильна відповідь

### Крок 9.7: Лідерборд

Після результату — екран рейтингу з твоєю позицією і балами.

### Крок 9.8: Повтори

Далі або наступний вибір категорії, або наступне питання — залежно від структури квізу. Цикл повторюється до кінця квізу.

Коли всі питання закінчились:

## Частина 10: Якщо щось пішло не так (Troubleshooting)

### 🔴 Pi не запускається

<details>
<summary><b>Немає жодного світла (LED) на Pi</b></summary>

</details>

<details>

- Носій не читається
- Якщо microSD — витягни, встав знову, переконайся що клацнула
- Якщо USB — спробуй інший USB порт (синій USB 3.0)
- Якщо USB і USB-завантаження не налаштоване — пройди Частину 2Б
- Якщо не допомагає — перезапиши носій (Частина 1)
</details>

<details>
<summary><b>Зелений LED блимає, але нічого на моніторі</b></summary>

- Перевір HDMI кабель: з'єднаний з портом HDMI 0 на Pi (ближче до живлення)?
- Перевір що монітор увімкнений і вибрано правильний HDMI вхід
- Спробуй інший HDMI кабель
</details>

### 🔵 Проблеми з USB-завантаженням (флешка / SSD)

<details>
<summary><b>Pi не завантажується з USB (тільки червоний LED)</b></summary>

Найімовірніша причина — USB boot не налаштований в EEPROM. Рішення:
1. Встав microSD з ОС → завантаж Pi
2. Відкрий термінал → `sudo raspi-config`
3. 6 Advanced Options → A4 Boot Order → B2 USB Boot
4. Finish → Reboot
5. Вимкни → витягни microSD → підключи USB → увімкни

- Перевір що SD-слот порожній (Pi завантажиться з SD замість USB якщо картка вставлена)
- Спробуй інший USB порт (синій USB 3.0)
</details>

<details>
<summary><b>Зелений LED блимає 4 рази і зупиняється (USB носій)</b></summary>

Pi не може знайти операційну систему на USB. Можливі причини:
- **USB boot не налаштований** — потрібно пройти Частину 2Б з microSD карткою
- **ОС не записана на USB** — перевір, що ти записував ОС саме на цей носій (Частина 1)
- **USB флешка/SSD в чорному порті** — спробуй синій порт USB 3.0
- **Несумісний USB носій** — деякі дешеві флешки не підтримуються. Спробуй іншу
</details>

<details>
<summary><b>Pi завантажується з SD замість USB (хоча USB носій підключений)</b></summary>

Це нормальна поведінка! MicroSD має вищий пріоритет за USB. Якщо хочеш завантажитись з USB — **витягни microSD** перед увімкненням Pi.
</details>

<details>
<summary><b>Pi завантажується з USB, але дуже повільно</b></summary>

- **USB флешка в чорному порті (USB 2.0)** — перестав у синій порт (USB 3.0)
- **Повільна USB флешка** — деякі флешки мають USB 3.0 конектор, але внутрішня швидкість як у USB 2.0. Спробуй іншу флешку або перейди на SSD
- **USB SSD через хаб** — підключай SSD напряму в Pi, без USB хабів
- Перевір що це USB 3.0 флешка (зазвичай має синій внутрішній конектор)
</details>

<details>
<summary><b><code>vcgencmd bootloader_config</code> показує BOOT_ORDER без 4</b></summary>

Значення BOOT_ORDER має містити `4` (USB). Наприклад: `BOOT_ORDER=0xf41` означає: спочатку USB (4), потім SD (1), потім перезапуск циклу (f). Якщо `4` немає — повтори Частину 2Б.
</details>

<details>
<summary><b>USB SSD не визначається (немає в <code>lsblk</code>)</b></summary>

- Перевір кабель — спробуй інший USB кабель
- Спробуй інший USB порт (синій USB 3.0)
- Деякі SSD потребують більше живлення — переконайся що блок живлення Pi 27W (не менше)
- Спробуй підключити SSD без інших USB пристроїв (тимчасово від'єднай клавіатуру і мишку)
- Спробуй підключити SSD до Mac — якщо Mac його теж не бачить, можливо SSD несправний
</details>

## Частина 11: Вимкнення (Clean Shutdown)

⏱ **Час: 5 хвилин**

Правильний порядок вимкнення:

### 1. Зупини GPIO сервіс
В SSH вікні де працює GPIO сервіс — натисни `Ctrl+C`:
```
^C
2026-03-21 [GPIO] Зупинка GPIO сервісу
```

### 2. Закрий Chromium на Pi
Якщо відкритий: `Alt+F4` або закрий вікно мишкою.

### 3. Вимкни Pi правильно

⚠️ **НЕ витягуй живлення просто так!** Це як висмикнути шнур з комп'ютера під час роботи — може пошкодити файли на носії (SD картці, USB флешці або SSD).

Правильний спосіб — через SSH:
```bash
sudo shutdown now
```

Або з клавіатури Pi:
```bash
sudo shutdown now
```

Дочекайся поки зелений LED на Pi перестане блимати (5–10 секунд). Тепер можна від'єднати живлення.

### 4. Зупини сервер на Mac
В Terminal з сервером натисни `Ctrl+C`:
```
Зупинка сервера (SIGINT)...
Сервер зупинено
```

### 5. Готово
Тепер все можна безпечно від'єднати: кабелі, монітор, кнопки.

## Частина 12: Резервна копія носія (Backup)

⏱ **Час: 20–30 хвилин** (більшість — очікування копіювання)

Тепер, коли все працює — зроби копію свого носія. Якщо щось зламається в майбутньому, ти просто запишеш копію назад і не потрібно повторювати весь цей гайд.

### Крок 12.1: Вимкни Pi і від'єднай носій

1. Вимкни Pi правильно: `sudo shutdown now`
2. Дочекайся поки LED перестане блимати (5–10 секунд)
3. Від'єднай живлення
4. Від'єднай носій:
   - **microSD:** натисни злегка на картку — вона вискочить із слоту знизу Pi
   - **USB флешка:** витягни флешку з USB порту Pi
   - **USB SSD:** від'єднай кабель від Pi (або витягни кабель з SSD)

### Крок 12.2: Підключи носій до Mac

- **microSD:** встав в Mac через адаптер (маленька картка → адаптер SD → слот Mac або USB кард-рідер)
- **USB флешка:** встав в будь-який USB порт на Mac
- **USB SSD:** підключи кабелем до USB порту Mac

### Крок 12.3: Знайди диск

В Terminal на Mac:
```bash
diskutil list
```

Знайди свій носій за розміром (наприклад, 32 GB для SD/флешки, 120 GB для SSD). Він буде щось типу `/dev/disk4`. Запиши номер диску.

Підказки для ідентифікації:
- **microSD:** зазвичай показується як "Generic STORAGE DEVICE MEDIA" або назва виробника
- **USB флешка:** назва виробника + розмір (наприклад, "Kingston DataTraveler")
- **USB SSD:** назва моделі (наприклад, "Samsung T7", "SanDisk Extreme")

⚠️⚠️⚠️ **ДУЖЕ ВАЖЛИВО:** переконайся що ти вибрав правильний диск! Якщо вкажеш неправильний диск в команді нижче — ти **зітреш свій Mac**. Перевір **тричі**. Диск Mac зазвичай `/dev/disk0` або `/dev/disk1` і набагато більший за розміром.

### Крок 12.4: Створи копію

Заміни `diskN` на номер твого диска (наприклад, `disk4`):
```bash
sudo dd if=/dev/rdiskN of=~/Desktop/pi-quiz-backup.img bs=4m status=progress
```

Пояснення команди:
- `sudo` — запуск з правами адміністратора (запитає пароль Mac)
- `dd` — програма копіювання
- `if=/dev/rdiskN` — **i**nput **f**ile: звідки читати (твій носій). `r` перед `disk` = raw mode (швидше)
- `of=~/Desktop/pi-quiz-backup.img` — **o**utput **f**ile: куди зберегти (файл на робочому столі)
- `bs=4m` — розмір блоку (4 мегабайти за раз — оптимально для швидкості)
- `status=progress` — показувати прогрес

Час копіювання залежить від розміру носія:
- **microSD 32 ГБ:** ~10–20 хвилин
- **USB флешка 32 ГБ:** ~10–15 хвилин
- **USB SSD 128 ГБ:** ~15–30 хвилин

⚠️ Файл резервної копії буде займати стільки ж місця на Mac, скільки весь носій (навіть якщо на носії використано тільки частину простору). Переконайся що на Mac достатньо вільного місця!

✅ **Що ти маєш бачити:** прогрес у мегабайтах, а наприкінці — повідомлення типу:
```
31914983424 bytes transferred in 580.123456 secs (55020123 bytes/sec)
```

### Крок 12.5: Безпечно від'єднай носій

```bash
diskutil eject /dev/diskN
```

Дочекайся повідомлення, потім фізично від'єднай:
- **microSD:** витягни з адаптера/кард-рідера
- **USB флешка:** витягни з USB порту
- **USB SSD:** від'єднай кабель

### Як відновити з резервної копії

Якщо колись потрібно буде відновити — та ж команда, тільки `if` і `of` міняються місцями:

1. Підключи носій до Mac (точно так як в Кроку 12.2)
2. Знайди диск: `diskutil list` (точно так як в Кроку 12.3)
3. Запиши копію назад:
```bash
sudo dd if=~/Desktop/pi-quiz-backup.img of=/dev/rdiskN bs=4m status=progress
```
4. Безпечно від'єднай: `diskutil eject /dev/diskN`

⚠️ Це зітре все що зараз на носії і замінить вмістом резервної копії!

⚠️ **Для USB SSD великого розміру:** якщо твій SSD 128 ГБ, а резервна копія зроблена з SD картки 32 ГБ — образ запишеться нормально, але Pi "бачитиме" тільки 32 ГБ. Щоб використати весь простір, після відновлення завантаж Pi і запусти `sudo raspi-config` → Advanced Options → Expand Filesystem.

### Клонування на новий Pi (для кількох подіумів)

Якщо хочеш зробити другий подіум — просто запиши бекап на новий носій того ж типу:

1. Підключи **новий** носій (порожній) до Mac
2. Знайди його в `diskutil list`
3. Запиши образ:
```bash
sudo dd if=~/Desktop/pi-quiz-backup.img of=/dev/rdiskN bs=4m status=progress
```
4. Встав у новий Pi → увімкни → все вже налаштовано!

⚠️ Після клонування зміни hostname нового Pi (щоб в мережі не було двох Pi з однаковим ім'ям):
```bash
sudo raspi-config
```
→ 1 System Options → S4 Hostname → введи нове ім'я (наприклад `pi-podium-2`)

⚠️ Якщо новий Pi завантажується з USB (флешка/SSD) — не забудь пройти Частину 2Б на кожному **новому фізичному Pi**. EEPROM налаштування зберігаються в чіпі самого Pi, а не на носії — клонування носія НЕ копіює ці налаштування.

Замість звичайного монітора + клавіатури — використати сенсорний екран. Тоді клавіатура і мишка не потрібні взагалі — гравець вводить нікнейм і вибирає відповіді пальцем.

### 📺 Другий монітор — SideMonitor
Pi 5 має два HDMI порти. Другий порт може показувати **SideMonitor** (`#/side`) — великий нікнейм гравця, видимий іншим учасникам. Скрипт `kiosk.sh` налаштований для обох дисплеїв.

### 🔄 Автозапуск
Повний скрипт `install.sh` налаштовує автозапуск: Pi вмикається → автоматично входить в систему → запускає Chromium в режимі кіоску → запускає GPIO сервіс. Жодних дій вручну.

### 📋 Кілька подіумів
Зроби резервну копію носія (Частина 12) і запиши її на нові носії для інших Pi. Кожен Pi стає окремим подіумом. Зміни hostname (`podium-1`, `podium-2`, ...) для ідентифікації. Для постійних подіумів рекомендується USB SSD — надійніший і швидший за microSD.

### 📺 Проекторний екран
Для аудиторії: відкрий `http://MAC_IP:8080/#/screen` на окремому комп'ютері або проекторі. Це показує питання, відповіді і рахунок на великому екрані для глядачів.

### 💾 Перехід на USB SSD
Якщо починав з microSD, можна мігрувати на SSD для швидкості та надійності (Частина 2Б + Частина 1).

## ⚙️ Примітки автора

Рішення, прийняті при написанні цього гайду:

1. **Три варіанти носія.** Гайд описує всі три варіанти: microSD (найпростіший), USB флешка (золота середина), USB SSD (найнадійніший для постійного подіуму). Вибір залежить від сценарію використання і бюджету.

2. **Ручний запуск Chromium замість kiosk.sh.** Скрипт `kiosk.sh` налаштований для двох дисплеїв і локального сервера. Для тесту з одним монітором і віддаленим сервером на Mac простіше відкрити браузер вручну.

3. **Python залежності без venv.** Рекомендація використовувати `--break-system-packages` замість virtualenv — для простоти. Для продакшн-подіуму краще використовувати venv.

4. **rpi-lgpio замість RPi.GPIO.** Pi 5 використовує чіп RP1 (не BCM2711 як Pi 4). Класична бібліотека RPi.GPIO не працює на Pi 5 — потрібен rpi-lgpio, який надає сумісний інтерфейс. В Pi OS Bookworm він встановлений за замовчуванням.

5. **GPIO пін 16 (фізичний) = GPIO23 (BCM) = Кнопка D.** Зверни увагу: фізичний пін 16 знаходиться в ПРАВІЙ колонці GPIO, на відміну від кнопок A/B/C, які всі в лівій. Це підтверджено в `gpio-service.py` (рядок 26) і `PODIUM_MANUAL.md`.

6. **Chromium через SSH.** Запуск GUI додатків через SSH на Pi OS Bookworm (Wayland) може бути проблематичним. Рекомендовано використовувати клавіатуру Pi для відкриття браузера. Варіант через SSH вказаний як альтернатива.

7. **Адреси магазинів.** Вказані реальні магазини (RPishop.cz, Alza.cz, Amazon.de, Conrad.de, BerryBase.de), але ціни є приблизними і можуть змінюватись.
